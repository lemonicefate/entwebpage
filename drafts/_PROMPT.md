#### Callout 語法（重要：使用 Obsidian 原生格式）

在適當段落插入 callout 區塊，轉換器會自動轉為網站的 NHS 風格色塊：

```markdown
> [!recommend] 建議
> 推薦病患採取的行動或好的做法

> [!caution] 注意
> 需要特別留意的事項或常見誤解

> [!danger] 警告
> 緊急或危險的警訊，需立即處理

> [!info] 小知識
> 補充說明的知識或有趣的醫學事實
```

**使用時機：**
- 居家照護的關鍵建議 → `[!recommend]`
- 用藥注意事項、常見誤解 → `[!caution]`
- 需立即就醫的警訊 → `[!danger]`
- 病患常有的疑問、醫學小知識 → `[!info]`

每篇文章至少使用 **2-3 個 callout**，分散在不同段落中。

#### Mermaid 流程圖

每篇至少 1 個流程圖，放在以下位置：
1. **就醫決策流程** — 放在「醫師怎麼幫你檢查」或「什麼時候該看醫生」段落
2. **治療階梯流程** — 放在「治療方式」段落末尾

格式使用標準 Mermaid（不使用自訂 CSS class）：

```markdown
 ```mermaid
flowchart TD
    A[症狀出現] --> B{持續超過 X 天？}
    B -->|是| C[建議就醫]
    B -->|否| D[居家觀察]
 ```
```

**整合診所服務時：** 在流程圖下方用 blockquote 說明診所優勢，不要在 Mermaid 語法中使用 `:::clinicService` 等自訂樣式。

#### 圖片預留位

在需要解剖示意圖或衛教圖片的位置插入：
```markdown
<!-- IMAGE_PLACEHOLDER: {描述} -->
```

#### 寫作規範

- **語氣**：專業但親切的醫師在診間對病患說話，用「你」不用「您」
- **術語**：首次出現必須括號補充白話解釋
- **段落**：每段不超過 4 句話，方便手機閱讀
- **Heading**：正文 heading 使用純文字，不加 emoji
- **禁止**：不推薦特定品牌藥物、不做診斷承諾、不使用恐嚇式行銷語言
- **醫療法第 85 條**：服務描述以「功能差異」為主，不做「效果承諾」，不使用「保證」「最好」「根治」等用語
